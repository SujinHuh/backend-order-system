# PR Template

## 변경 사항
**PR 제목:** `감사 follow-up 문서-구현 정합성 보강`

- [ ] 신규 기능 추가
- [x] 버그 수정
- [x] 테스트 보강
- [x] 문서 업데이트

## 변경 내용 요약
- 상품명 최대 100자 요구사항을 엔티티 컬럼 길이와 API validation에 반영
- `Product` 감사 필드의 not-null 제약을 문서의 DB 모델 요구사항에 맞게 보강
- 주문 상태/생성일시, 주문 항목의 주문/상품 FK 조회 인덱스를 엔티티에 반영
- 상품 재고 추가/차감 음수 입력 방어 테스트 추가
- 상품명 100자 초과 요청의 400 응답 컨트롤러 테스트 추가
- 완료 주문에 동시 취소 요청이 들어와도 재고가 중복 복구되지 않는 동시성 테스트 추가
- README의 상품/주문 목록 조회 API에 `sort` query parameter 예시 추가

## 변경 이유
- 감사 결과, 문서에 정의된 상품명 길이 제한과 DB 제약 일부가 실제 구현에 완전히 반영되어 있지 않았음
- 문서의 성능 고려 사항에 명시된 조회 인덱스가 JPA schema 생성 경로에 누락되어 있었음
- 주요 비즈니스 규칙 중 재고 음수 입력 방어와 완료 주문 동시 취소 시 중복 복구 방지 테스트 근거가 부족했음
- README가 실제 `Pageable` 기반 정렬 지원을 설명하지 않아 API 문서와 구현 사이에 작은 불일치가 있었음

## 테스트 및 검증

- README의 상품/주문 목록 조회 예시가 실제 페이징/정렬 지원과 맞는지 확인

- 주문 상태 정책, 재고 차감/복구 시점, 비관적 락 기반 동시성 전략은 기존 decision을 변경하지 않고 유지함
