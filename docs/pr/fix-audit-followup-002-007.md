## 변경 내용 요약
- 상품 등록/수정 요청의 `stockQuantity` 누락 검증을 위해 DTO 필드를 `Integer`로 변경하고 `@NotNull` 추가
- 상품 등록/수정 API에서 `stockQuantity` 누락 시 400 응답 테스트 추가
- 상품 서비스의 실제 H2/JPA 기반 저장, 조회, 수정 통합 테스트 추가
- 주문 생성 API의 빈 항목, 수량 오류, 상품 미존재 실패 케이스 컨트롤러 테스트 추가
- 주문 상태 변경 API의 잘못된 상태 전이 400 응답 컨트롤러 테스트 추가
- 002~007 태스크 문서의 close-out 메타데이터와 감사 follow-up 검증 기록 정리

## 변경 이유
- 완료 태스크 감사 결과, 일부 문서가 주장한 API 실패 경로와 DB 연동 검증 증거가 현재 테스트셋에 충분히 보존되어 있지 않았음
- `stockQuantity`가 primitive `int`라 요청에서 필드가 누락돼도 기본값 `0`으로 처리될 수 있어 필수 입력 검증을 명확히 할 필요가 있었음
- 완료된 태스크의 `phase_status.md` 일부가 close-out 상태와 맞지 않는 stale metadata를 포함하고 있어 문서 정합성 보강이 필요했음
