# Issue

## 배경

작은 문서 정렬 작업에도 경량 운영 예외를 어떻게 판단하고 기록하는지 예시가 필요하다.

## 요청사항

- `docs/project/standards/implementation_order.md`의 문장 하나를 더 명확한 표현으로 정리한다.
- 경량 운영 예외를 적용한 판단 근거와 유지한 절차를 작업 산출물에 남긴다.

## 비범위

- project overlay 구조 변경
- 공통 phase 규칙 변경
- maintainer 전용 문서 변경

## 승인 또는 제약 조건

- 변경은 문장 명확화 범위에 한정한다.
- 외부 계약, 데이터 스키마, 템플릿 구조 변경으로 확장되면 표준 Phase 흐름으로 전환한다.
- 사용자 승인 후에만 경량 운영 예외를 적용한다.
