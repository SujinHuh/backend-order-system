# Issue

## 제목

주문 시스템 API 서버 구현

## 배경

개발 3팀 backend 개발자 채용 과제로 상품과 주문 도메인을 기반으로 한 REST API 서버를 구현한다.

본 과제는 단순 CRUD뿐 아니라 주문 상태 전이, 재고 정합성, 트랜잭션, 동시성 처리, 예외 응답, 테스트, 성능 고려사항을 함께 평가한다.

## 요청사항

- Spring Boot 3 기반 API 서버를 구현한다.
- Java 17 이상을 사용한다.
- Maven을 사용한다.
- H2 Database를 사용한다.
- JPA와 QueryDSL을 사용한다.
- 상품 도메인 API를 구현한다.
- 주문 도메인 API를 구현한다.
- 주문 상태 변경과 재고 차감/복구 정책을 구현한다.
- 동시 주문 상황에서도 재고 정합성을 보장한다.
- 주요 비즈니스 로직 테스트를 작성한다.
- README에 실행 방법, 설계, 동시성 처리, 성능 고려사항을 정리한다.

## 제약사항

- 과제 내용은 외부 유출하지 않는다.
- 미완성인 경우에도 제출 가능하지만, 구현 범위와 미완성 범위는 명확히 남긴다.
- 제출 전 README에 지원자 성명을 포함한다.
- 제출 저장소는 private repository로 관리한다.

## 비범위

- 결제, 배송, 회원, 인증/인가 기능
- 실제 운영 DB 연동
- 외부 API 연동
- 프론트엔드
- 대규모 부하 테스트 자동화
