# Issue

## 배경

`docs/final-readme` 브랜치는 구현 완료된 주문 시스템의 최종 README를 실제 코드와 task 산출물 기준으로 정리하기 위한 작업이다.

현재 `README.md`는 과제 개요, 도메인 설계, API 설계 방향, 실행 방법을 포함하지만 일부 문구가 구현 전 설계/예상 상태에 머물러 있다. 또한 001~008 task를 거치며 실제 구현된 공통 예외, 상품 API, 주문 생성, 상태 변경/재고 정책, 동시성 제어, QueryDSL 조회 API, 테스트/검증 결과를 최종 제출 문서 관점에서 정리해야 한다.

## 요청사항

- `main` 최신 코드에서 `docs/final-readme` 브랜치를 기준으로 최종 README 정리를 진행한다.
- `README.md`를 실제 구현 기준으로 갱신한다.
- 완료된 task 문서와 실제 코드가 말하는 범위를 README에 반영한다.
- 실행 방법, 테스트 방법, 주요 API, 도메인 정책, 트랜잭션/동시성 정책, 조회 전략, 예외 응답, 검증 범위를 제출자가 이해할 수 있게 정리한다.
- 문서가 실제 구현과 어긋나거나 과장하지 않도록 한다.

## 비범위

- 애플리케이션 기능 코드는 변경하지 않는다.
- 테스트 코드는 변경하지 않는다.
- 새로운 API, 예외 코드, validation 정책, DB migration, Testcontainers, 운영 DB 설정을 추가하지 않는다.
- 기존 상태 전이, 재고 차감/복구, 동시성 제어 정책을 변경하지 않는다.
- `docs/project/decisions/*`는 새 장기 정책 결정이 필요한 경우가 아니면 수정하지 않는다.

## 승인 또는 제약 조건

- README는 `main`의 실제 구현과 통과한 검증 결과를 기준으로 작성한다.
- 문서에 적는 API와 정책은 controller/service/domain/repository 구현 및 task 산출물로 확인 가능한 범위만 포함한다.
- 구현되지 않은 운영 기능이나 성능 보장을 완료된 것처럼 적지 않는다.
- 최종 검증에는 최소 `./mvnw test`, `git diff --check`, phase gate 검증을 포함한다.
