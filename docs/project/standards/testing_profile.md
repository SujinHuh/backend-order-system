# Testing Profile

## 목적

이 문서는 주문 시스템 API 서버의 테스트 전략과 테스트 범위를 정의한다.

테스트 실행 명령, 필수 여부, 실패 처리 기준은 `docs/project/standards/quality_gate_profile.md`에서 정의한다.

## 테스트 전략

- 도메인 규칙은 단위 테스트로 검증한다.
- Service 유스케이스는 Spring 통합 테스트로 검증한다.
- Repository의 QueryDSL 조건 조회와 lock 조회는 JPA 통합 테스트로 검증한다.
- Controller는 필요 시 MockMvc 통합 테스트로 요청/응답 계약을 검증한다.

## 테스트 환경

- 기본 테스트 DB는 H2를 사용한다.
- 외부 서비스 의존성은 없다.
- Testcontainers는 필수로 사용하지 않는다.
- 테스트 데이터는 각 테스트에서 직접 생성한다.

## quality_gate_profile에 두는 항목

- test 실행 명령
- test 게이트의 필수 여부
- 언제 test를 반드시 돌려야 하는지
- 테스트 실패 시 커밋/승인 불가 기준

## 핵심 테스트 대상

- 상품 등록 성공
- 상품 수정 성공
- 상품 단건 조회 성공
- 상품 목록 페이징 조회 성공
- 카테고리 기준 상품 조회 성공
- 주문 생성 성공
- 주문 생성 시 기본 상태가 대기인지 검증
- 존재하지 않는 상품으로 주문 생성 실패
- 주문 항목이 비어 있는 주문 생성 실패
- 0 이하 수량 주문 실패
- 허용된 주문 상태 변경 성공
- 허용되지 않은 주문 상태 변경 실패
- 주문 완료 시 재고 차감
- 재고 부족 시 주문 완료 실패
- 주문 취소 시 차감된 재고 복구
- 동시에 주문 완료 요청이 발생해도 재고가 음수가 되지 않는지 검증

## 동시성 테스트

동일 상품 재고보다 많은 주문 완료 요청을 동시에 발생시켜 다음을 검증한다.

- 성공한 주문 수만큼만 재고가 차감된다.
- 실패한 주문은 재고 부족 예외로 처리된다.
- 최종 재고는 음수가 되지 않는다.

## Coverage 기준

정량 coverage 기준은 강제하지 않는다.

대신 다음 비즈니스 핵심 경로는 테스트가 없으면 Phase 3 종료 불가로 본다.

- 주문 상태 전이
- 재고 차감
- 재고 부족 예외
- 존재하지 않는 상품 예외
- 동시 주문 완료 시 재고 정합성

## 테스트 실행

```bash
./mvnw test
```
