# 과제 내용

## 1. 설계 및 구현

- 주어진 요구사항을 기반으로 백엔드 시스템을 설계하고 구현하세요.
- H2 Database를 사용하여 데이터베이스 모델을 설계하세요.
- 상품, 주문 도메인에 대해 필요한 RESTFUL API 엔드포인트를 정의하고 구현하세요.
- 상품 목록 조회, 주문 생성, 주문 상태 변경 등 핵심 기능을 포함해야 합니다.
- 각 API에 대해 예상되는 요청(Request) 및 응답(Response) 구조를 자유롭게 정의하세요.
- 도메인 데이터는 정형화 되도록 설계하세요.
- 유연한 비즈니스 모델링으로 구현하세요.

---

## 2. 데이터 무결성 및 예외 처리

- 잘못된 요청에 대해 적절한 예외 처리와 일관된 에러 응답을 제공해주세요.
- 다음과 같은 예외 상황을 반드시 고려해주세요.
    - 존재하지 않는 상품에 대한 주문 요청
    - 재고 수량이 부족한 상품 주문
    - 허용되지 않은 주문 상태 변경 요청
- 데이터 정합성이 깨지지 않도록 트랜잭션 처리 방안을 고려해주세요.

---

## 3. 동시성 및 트랜잭션 고려

- 여러 주문이 동시에 발생하더라도 상품 재고가 정확하게 차감되도록 구현해주세요.
- 동시성 문제를 어떻게 해결했는지 코드 또는 README에 간단히 설명해주세요.
    - 예시:
        - 락 사용
        - 트랜잭션 전략
        - 기타 접근 방식 등

---

## 4. 테스트

- 주요 비즈니스 로직에 대해 테스트 코드를 작성해주세요.
- 단위 테스트 또는 통합 테스트 중 자유롭게 선택할 수 있습니다.
- 테스트 대상과 테스트 의도가 코드 또는 설명을 통해 명확히 드러나야 합니다.

## 5. 성능 고려

- 주문 처리 과정에서 발생할 수 있는 성능 이슈를 고려하여 구현해주세요.
- 주문 데이터가 많아질 경우 발생할 수 있는 문제점을 분석하고, 이에 대한 개선 방안을 제시하세요.
